## What is Fleetarr?

Fleetarr gives you an at-a-glance status and activity dashboard across your self-hosted media
automation stack. See what's healthy, what needs attention, and clear the small, frequent problems
right from your phone.

**Supported services:** Sonarr · Radarr · Prowlarr · Seerr · SABnzbd · Plex · Jellyfin

## Features

- 🩺 **Combined fleet health** — a single at-a-glance problem count across every service
- 📊 **Per-service detail** — download queues, indexer status, pending requests, active streams, calendar and history
- ⚡ **Quick actions** — approve a request, remove a stuck download, stop a runaway stream, retry a failed grab
- 🔒 **Works on your network** — Tailscale / WireGuard / LAN, with self-signed certs and reverse-proxy sub-paths
- ☁️ **Syncs across your devices** — via iCloud, end-to-end encrypted; keys stay in iCloud Keychain
- 🍎 **Native and adaptive** — light & dark, Dynamic Type, VoiceOver, Home Screen widget and macOS menu bar

## Privacy

Fleetarr talks only to your own services and keeps its configuration on your own devices. Your API
keys stay in iCloud Keychain (end-to-end encrypted), and usage analytics are fully anonymous. See
the [privacy policy](https://radaiko.github.io/Fleetarr/privacy.html).

## Architecture

```
Fleetarr.xcodeproj   generated by xcodegen from project.yml (do not edit by hand)
App/                 the SwiftUI multiplatform app target (views, SwiftData store, app wiring)
FleetarrKit/         local Swift package: models, networking, per-service API clients, view models
```

- **`FleetarrKit`** is UI-free and fully unit-testable from the command line. It holds the domain
  models, the `FleetService` protocol, the concurrent networking layer, and one API client per
  service (`Sonarr`, `Radarr`, `Prowlarr`, `Seerr`, `SABnzbd`, `Plex`, `Jellyfin`). Built in
  Swift 6 language mode (strict concurrency).
- **`App`** is the thin SwiftUI layer: adaptive navigation, the Fleet dashboard, instance
  management, and the SwiftData + CloudKit persistence for non-secret config. Credentials live in
  iCloud Keychain (`FleetarrKit`'s `KeychainStore`), never in the SwiftData store.

## Building

Requires **Xcode 26+** and **xcodegen** (`brew install xcodegen`).

```sh
# Generate the Xcode project from project.yml
xcodegen generate

# Run the FleetarrKit unit tests (no Xcode project needed)
cd FleetarrKit && swift test

# Build the app for macOS (unsigned, for CI/local validation)
xcodebuild -project Fleetarr.xcodeproj -scheme Fleetarr \
  -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO build
```

### Enabling iCloud sync (requires a signing team)

The SwiftData store syncs non-secret config via CloudKit and credentials via iCloud Keychain.
To enable, open the generated project in Xcode with your Apple Developer team and turn on the
**iCloud (CloudKit)**, **Keychain Sharing**, and **Background Modes → Remote notifications**
capabilities. `App/Fleetarr.entitlements` documents the expected entitlements.
